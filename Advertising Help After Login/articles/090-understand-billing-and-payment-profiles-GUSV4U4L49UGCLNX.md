# Understand Billing and Payment Profiles

This article explains global billing and payment profiles for Amazon Ads.

## Billing Profiles

A billing profile defines who pays for advertising and contains billing address, tax details, and contact information. It is used for tax calculation and commercial invoices.

| Account type | Required information |
| --- | --- |
| Advertiser / brand owner | Advertiser name, address, emails, optional billing address, and VAT number if applicable. |
| Agency | Agency name, holding company if applicable, address, email address, optional billing address, and VAT number if applicable. |

Purchase order or booking reference numbers can be added to a billing profile and continue appearing on invoices until updated.

## Global Billing Profiles

- Global billing profiles copy existing billing information when setting up accounts in new countries.
- Administrators can manage billing profiles from one page.
- Profiles can include billing addresses, tax IDs, payment methods, and purchase order numbers.
- A default profile can be applied automatically to newly registered countries.
- If additional country-specific information is needed, Amazon Ads prompts for missing details.

## Global Payment Profiles

- Global payment profiles copy existing payment methods when setting up new countries.
- Administrators can manage payment profiles from one page.
- A default payment profile can be applied to newly registered countries.

## Checkpoints

- Stop before changing billing, tax, PO, or payment profile details.
- Confirm whether the account is an advertiser/brand owner or agency.
- Confirm country-specific tax and billing requirements before saving.
