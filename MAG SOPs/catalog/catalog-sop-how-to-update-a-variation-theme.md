# Catalog SOP: How to Update a Variation Theme

Source: https://sop.myamazonguy.com/books/catalog/page/catalog-sop-how-to-update-a-variation-theme

## **How to Update a Variation Theme**

**Who is this for :** This is for the Account Specialists who need to update the variation theme of a parentage.

**Objective :** This is to provide the Account Specialists with the necessary steps and guidelines to efficiently and accurately update the variation theme of a parentage.

**Instructions:**

**Part 1 : Backup file**

- Ensure that you obtain the latest version of the Category Listings Report. Attach it to both the task in your Asana platform and the client's folder.

Step 1. Create your sheet/tracker of the existing parentage.

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/6pvimage.png)

Step 2. Download the "Category Specific Template" for the parentage.

**Part 2 : Fill out the flat file.**

- The main goal is to re-upload the current parentage, but this time with its newly assigned theme variation.
- Since we already have the tracker of the parentage, open the template we just downloaded and fill out the necessary information.

**NOTE:** Only provide the essential information. It is the same procedure outlined in creating a parentage.

**Parent Listing**

- Product Type
- Parent SKU
- Parent ASIN
- Brand Name
- Parent Name
- Item Type Keyword
- Variation Data

  - Parent SKU
  - Variation Theme (enter new proposed variation theme)
- Update_Delete

**IMPORTANT NOTE:** To ensure that the newly assigned variation theme is immediately reflected in the entire parentage, label the parent row as **"Update."**

- Item Condition

**NOTE:** Please ensure that the bullets, descriptions, and search terms saved on the backend are included when the Parent listing is fully optimized.

**Child Listings**

- Child SKU/s
- Child ASIN/s
- Variation Data

  - Variation Theme (enter new proposed variation theme)
  - Parent SKU
  - Parentage
  - Relationship Type
- Variation Theme Name
- Update_Delete

IMPORTANT NOTE: The rest of the child ASINs are in**"Partial Update"**

- Fulfillment Center ID

**REMINDER:**

When you submit inventory feeds to update inventory levels for items that you sell on Amazon's website, never leave the fulfillment-center-id null. For a self-fulfilled inventory, enter DEFAULT in this column. For Fulfillment by Amazon inventory, enter either AMAZON_NA, AMAZON_EU, AMAZON_JP, or AMAZON_IN depending on your marketplace. This helps ensure that each item is fulfilled as you expect it to be.

**NOTE:**

- If it’s FBA, you have to fill out the following columns:

  - “batteries_required”
  - “supplier_declared_dg_hz_regulation1” under the Compliance section.
  - Pesticide Marking

    - EPA Registration Number
  - Pesticide Registration Status

    - This product is not a pesticide or pesticide device, as defined under the U.S. Federal Insecticide, Fungicide, and Rodenticide Act.
- **Sample scenario:**Update the variation theme from Scent to Size-Scent.

**Part 3 :** Save the flat file and upload it on Seller Central.

- Go to Catalog > Add Products via Upload > Submit Products. Refresh the page first before attaching the file, and make sure you’re on the right Amazon store. Then click "Submit Products."

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-1712150144408.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/FXVimage.png)

- Once the file has been uploaded and processed, let it sit for 15 minutes. It can take a little time for the file to finish being processed.

**Important Note:**

- Monitor the Upload Status. If it says “Complete Drafts”, download the Processing Report to see what caused the error and fix it accordingly.

**Part 4 :** Verify the updated Variation Theme of the Parentage.

- Once you receive the update from Amazon about the successful processing of your upload file, find the parentage in the catalog and check if the variation theme was successfully updated on both the backend and front end.
- You can verify it using these options:

**Manage All Inventory**

- Go to Seller Central > Inventory > Manage All Inventory and search for the Parent SKU or ASIN.
- Make sure that the Status and Fulfilled by are set to “ALL”.
- Click the ‘Edit’ option of Parent SKU and look for the ‘Variations’ tab to view the current variation theme.

FAQs

**1. What is a variation theme, and why is it important?**

- A variation theme is the relationship that links parent and child products (e.g., Size, Color, Scent). It allows customers to easily navigate between options on the product detail page, improving the shopping experience and increasing sales potential.

**2. Why would I need to update a variation theme?**

- You might need to update a variation theme if:

  - The existing theme no longer reflects the product’s structure (e.g., moving from Scent to Size-Scent).
  - New child ASINs require a broader or more specific theme.
  - Amazon's policies or guidelines have changed.

**3. What is the first step before updating a variation theme?**

- Always backup the current Category Listings Report (CLR) to preserve the existing parentage information. Save it to both Asana and the client’s folder for future reference.

**4. What is the difference between “Update” and “Partial Update” in the flat file?**

- Update: Used for the parent SKU to ensure the new variation theme is applied to the entire parentage.
- Partial Update: Used for child ASINs to update only specific fields without overwriting existing data.

**5. Can I update multiple parentages simultaneously?**

- Yes, you can update multiple parentages in one flat file. Ensure each parentage is clearly organized and all required fields are completed for every parent and child ASIN.

**6. What should I do if I see red-highlighted fields in the flat file?**

- Red-highlighted fields indicate required attributes. If the necessary data isn’t readily available:

  - Check the Seller Central backend for the missing information.
  - Inform the POC (Point of Contact) if further input is needed.

**7. How do I fill out fulfillment-related columns in the flat file?**

- For fulfillment columns:

  - Self-fulfilled inventory: Use DEFAULT for fulfillment-center-id.
  - FBA inventory: Use AMAZON_NA, AMAZON_EU, AMAZON_JP, or AMAZON_IN based on your marketplace.

**8. What are the compliance columns I need to complete for FBA products?**

- For FBA items, ensure you fill out:

  - batteries_required: Indicate “Yes” or “No.”
  - supplier_declared_dg_hz_regulation1: Select the appropriate regulation status.
  - Pesticide Marking: Include EPA Registration details or confirm the product is not a pesticide.

**9. What happens if the uploaded flat file contains errors?**

- If errors occur:

  - Check the Processing Report under Upload Status.
  - Identify and resolve the errors.
  - Re-upload the corrected flat file.

**10. How do I verify if the variation theme update was successful?**

- You can verify the update using one of the following methods:

  - Manage All Inventory: Check the Variations tab of the Parent SKU in Seller Central.
  - Variation Wizard: Use the "Add to or update an existing variation family" option to review the updated theme.
  - Amazon Detail Page: View the Parent ASIN’s detail page to confirm the updated theme is live.

**11. Can I revert a variation theme update?**

- Yes, by uploading a new flat file with the original variation theme and structure. Ensure you have the backup CLR to recreate the previous parentage.

**12. What should I do if the updated theme doesn’t reflect after 24 hours?**

- If changes are not visible:

  - Verify that the flat file was successfully processed.
  - Check for backend errors in the Processing Report.
  - Contact Amazon Seller Support if the issue persists.

**13. What are some common errors when updating variation themes?**

- Common errors include:

  - Missing or incorrect Parent or Child SKUs.
  - Incomplete required fields (e.g., variation theme or fulfillment-center-id).
  - Incorrect use of “Update” and “Partial Update.”
  - Mismatched Parent-Child relationships.

**14. Can I update the variation theme for a single child ASIN without affecting the rest?**

- No, variation themes must be consistent across the entire parentage. Any change to the variation theme will apply to both the parent and all child ASINs.

**15. Are there limitations to which variation themes I can use?**

- Yes, available variation themes depend on the product category. Refer to the Category Specific Template to determine acceptable themes for your category.

**16. What should I include in the flat file to optimize the Parent SKU?**

- For the Parent SKU:

  - Fully optimized bullets, descriptions, and search terms.
  - Ensure all mandatory fields, such as Product Type, Brand Name, and Item Type Keyword, are completed.

**17. What happens if I leave the fulfillment-center-id column blank?**

- Leaving the fulfillment-center-id column blank can cause errors or prevent updates. Always use DEFAULT for self-fulfilled inventory and the appropriate identifier for FBA inventory.

**18. Can I add new child ASINs when updating the variation theme?**

- Yes, you can add new child ASINs during the update. Ensure they are included in the flat file under the updated variation theme with proper Parent-Child relationships defined.

**19. How do I handle ASINs with multiple variations when updating the theme?**

- Ensure all variations are included in the flat file and that the new theme accommodates the existing variations. Use the Variation Theme Name column to apply the updated theme.

**20. Will updating the variation theme impact the product detail page layout?**

- Yes, changing the variation theme may alter how options (e.g., Size, Color) appear on the product detail page. Always preview changes to ensure they align with the desired customer experience.
