# Catalog SOP: A/B Test (A+ Content)

Source: https://sop.myamazonguy.com/books/catalog/page/catalog-sop-ab-test-a-content

## **A/B Test (A+ Content)**

**Who is this for :** Individuals who wished to conduct A/B split testing within Seller Central.

**Objective :** The purpose of this document is to provide a step-by-step guide on how to create A/B split testing to submit alternate content for your products to collect insights such as traffic and conversion.

Split testing or AB testing, on Amazon, is an excellent way of optimizing your listings through experiments with product content, images, and pricing. Four crucial elements of the listing affect conversion rate: Image, Title, Product Features, Description, and Price.

**Seller Central A/B Testing**

Access the Amazon Seller Central page by clicking the [URL](https://sellercentral.amazon.com/)

- Go to Brands, then select Manage Experiments.

- On the experiments dashboard, click Create a New Experiment.

  - You can run an experiment for A+ Content, Bullet Points, Main Image, Product Description, and Product Title.

- Content Selection

  - "A" content

    - Must be previously approved
    - Must be associated with the Reference ASIN
    - Cannot be part of a current A/B test
    - Cannot have more than 500 associated ASINs
  - "B" content

    - Must be associated with the Reference ASIN
    - Cannot be part of a current A/B test
    - Will automatically inherit ASINs from A
    - If “B” content is newly drafted, add the ASIN into it and save it as Draft

FAQs

**1. What is A/B testing for A+ Content, and why should I use it?**

A/B testing for A+ Content allows you to experiment with different versions of your A+ Content to determine which version drives better performance in terms of traffic and conversion rates. By testing variations, you can gain insights into which design, messaging, or layout resonates better with customers, ultimately helping you optimize your product listings for improved sales performance.

**2. What are the prerequisites for conducting A/B testing on A+ Content?**

- Before you can set up A/B testing for A+ Content, the following requirements must be met:

  - Your A+ Content must be approved by Amazon.
  - It must be associated with the Reference ASIN for the test.
  - The A+ Content must not already be part of another active A/B test.
  - For "A" content, ensure it has fewer than 500 associated ASINs. If it has more, you may need to revise your setup.
  - There should be a draft or approved A+ Content already created before you begin the A/B testing process.

**3. How long does it take for A+ Content to get approved by Amazon?**

- A+ Content typically takes between 7 to 10 days for Amazon to review and approve. However, this timeline can vary depending on the complexity of the content and any potential issues Amazon encounters during review. If your A+ Content is not approved within this time frame, check your content for compliance with Amazon's guidelines.

**4. How do I create "B" content for A/B testing?**

- To create "B" content, you can either:

  - Draft new A+ Content: If you want to create a completely new version of the content, draft it and then save it as a draft in the A/B testing tool.
  - Modify existing A+ Content: You can modify your existing content and associate it with the same Reference ASIN. Just make sure that any changes align with your experiment goals.
  - Once you’ve created or modified the "B" content, make sure to save it as a Draft if it’s new, or associate it properly with the reference ASIN.

**5. What happens if I try to associate more than 500 ASINs with "A" content?**

- If you try to associate more than 500 ASINs with the "A" content, Amazon will not allow you to complete the A/B test setup. You will need to reduce the number of ASINs associated with the content or split them into smaller batches for testing. Ensure that the number of ASINs is within the limit before submitting the test.

**6. How do I know if my A/B test is set up correctly before submission?**

- Before you submit your A/B test, make sure you’ve reviewed the following:

  - Both A and B content are correctly associated with the Reference ASIN.
  - The duration, test name, and hypothesis are clearly defined.
  - The B content is saved as a draft (if newly created) or properly linked to the test.
  - Ensure there are fewer than 500 ASINs linked to the "A" content. Once you’ve confirmed all the details are correct, click Submit A/B Test and check the experiment status on the dashboard for any errors.

**7. What happens if I don’t see my A/B content options in the experiment setup?**

- If you don’t see the content options for A/B testing, make sure the following:

  - A+ Content has been approved and associated with the reference ASIN.
  - There is no ongoing A/B test on that content.
  - Your "A" content is properly linked and has fewer than 500 ASINs. If everything seems in order but you still don’t see the content options, try refreshing the page or contacting Seller Support for further assistance.

**8. How do I track and analyze the results of my A/B test?**

- After your A/B test has been completed, you can track the results by going to the Experiments Dashboard in Seller Central. Look at the traffic and conversion rate data for both versions of the content. Amazon will show you how each variant performed, so you can determine which one resulted in better customer engagement and sales performance.

**9. Can I run multiple A/B tests simultaneously on the same ASIN?**

- You can run multiple A/B tests, but be mindful of what you’re testing. For example, you can test A+ Content against Product Description or Main Image with the same ASIN, but you need to ensure that tests don’t overlap in ways that could distort results. Each test should be independent, and the content being tested should be clearly separated.

**10. What does it mean that "B" content automatically inherits ASINs from "A" content?**

- When you create "B" content for your A/B test, it automatically inherits the ASINs associated with the "A" content. This means that if "A" content is linked to multiple ASINs, the same ASINs will be included in the "B" content test by default. You can modify or remove ASINs if necessary, but it saves time by automatically associating them with the "B" content.
