# Catalog SOP: Parentage Creation

Source: https://sop.myamazonguy.com/books/catalog/page/catalog-sop-parentage-creation

## **Parentage Creation**

**Who is this for :**This is for the Account Specialists who are tasked to create a variation.

**Objective :** To make sure that Account Specialists are equipped with the knowledge to utilize the Category Specific template in creating product variations, which involves establishing connections between products on Amazon, such as connecting a variation to its parent product listing.

**Instructions:**

1. Download a category-specific template.
2. Identify the SKUs that you want to add to the parentage (both FBA and FBM)

We need to identify the SKUs that we will be adding to the parentage and ensure that both the FBA and FBM SKUs are added to the template upload file to make the catalog more organized.

a. When you have your complete list of ASINs that need to be parented, go to Inventory > Manage All Inventory

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/parentage-creation-image-dj03afnt.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/Pmjimage.png)

b. Search for each one in the catalog and grab the SKU of each item under that ASIN (make sure that the “Status” and “Fulfilled by” are set to “All”).

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-1680195600575.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/svmimage.png)

c. Identify the variation names to be used for the parentage based on the variation theme provided. (SKUs with the same ASIN will have the same variation name).

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-1680195849118.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/LMJimage.png)

d. Open the flat file and fill out the necessary information.

3. Fill out the data needed for Parent and Child ASINs.

**A. Parent Listing**

**I. Product Type**

This can be found in the first column of the flat file.

Reminder: Choosing the correct product type on a template, will give you valid values or dropdown options to choose from on some attributes.

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/parentage-creation-image-asszour7.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/gjzimage.png)

**II. Parent SKU Create a unique parent SKU by identifying the common elements between all of the child ASINs and adding the keyword "PARENT" Then add the new SKU to the template in the “Seller SKU” section for the Parent ASIN. For clients that have specific naming conventions, confirm if they have a Parent SKU preference.**

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-1680196157586.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/7qNimage.png)

**III. Brand Name**

The exact brand of the product. Populate this field so your brand name is displayed on the Amazon.com site.

**IMPORTANT**: Make sure that the brand name of the parent listing should match the child Asins' brand name.

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-1680196240188.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/yERimage.png)

**IV. Parent Title**

Sometimes the Parent SKU title is visible on the detail page so we need to make sure that the title is not variation or SKU-specific. Remove all of the terms that would identify the item as a specific variation.

Example:

Child Title:

This is the Way Wine or Coffee Tumbler 12 oz

Updated Parent Title:

This is the Way Wine or Coffee Tumbler

**V. Item Type Keyword**

Use this to specify what your item is. This field is primarily used for browse classification.

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/parentage-creation-image-ywas1rk0.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/clbimage.png)

**VI. Variation Data For Parent SKU:**

1. Parentage: Parent
2. Parent SKU: (Leave this blank)
3. Relationship Type: (Leave this blank)
4. Variation Theme: Select from the dropdown Ex. Size

NOTE: In the Variation theme field, enter the same value that you entered for Variation Theme on the parent (this value will be the same for all child items).

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-1704438447173.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/M3Limage.png)

**VII. Update_Delete**

Choose "Update" if you're in the process of creating the parent listing, or "PartialUpdate" if the parent SKU you want to use already exists.

You will need to find the attribute column that corresponds with the variation name. In this example, we are using "size name". Following this, locate the size column on the flat file and paste the size names of each child SKU.

Press CTRL + F to bring up the Find tool and enter “size” (or your variation theme” to find the size attribute.

**IV. Update_Delete**

**PartialUpdate**

This column should be filed with PartialUpdate for all child SKUs. We can use Update on the parent if the parent listing does not exist in the account yet, otherwise use PartialUpdate for the Parent Row. If you use “Update” instead of “PartialUpdate”, Amazon will likely erase the data for attributes that are left blank on the template.

Under the Fulfillment Center ID column, use AMAZON_NA (NA for North America, CA for Canada, MX for Mexico, etc) if it’s Fulfilled by Amazon and DEFAULT if it’s Fulfilled by Merchant.

For any seller-fulfilled inventory, enter DEFAULT in this column. For Fulfillment by Amazon inventory, enter either AMAZON_NA, AMAZON_EU, AMAZON_JP, or AMAZON_IN depending on your marketplace.

Manual Parentage Creation

### **Manual Parentage Creation**

**Overview**

Manual parentage creation is an alternative method to link child ASINs under a parent listing when the flat file method does not work. It is especially useful for troubleshooting errors or creating a small number of parent-child variations.

**Things to Consider**

- This method is ideal if you're having trouble creating parentage via a feed file.
- You cannot add the same ASIN twice in a manual parentage setup.

Example: If both FBA and FBM listings exist for the same ASIN (e.g., same size), only one can be added manually due to identical variation values. Add the second via feed file after the parentage is created.

- Amazon will automatically assign the variation theme based on the product category.
- Amazon will change the child product titles to match the parent title upon creation. Be sure to save a copy of each product title before proceeding. You can revert titles after parentage is created (either manually or via feed file).

**Example Scenario**

You are creating parentage between two listings: **ASIN1** and **ASIN2**.

**Step-by-Step Procedure**

1. **Go to Manage Inventory**

  - Locate **ASIN1** and click **Edit Listing**.

    ![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-R633AYNV.png)

    ![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/eXJimage.png)
2. **Navigate to the Variations Tab**

  - Click on the **“Variations”** tab in the edit listing menu.

    ![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-R8FM8V2F.png)

    ![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/lZaimage.png)
3. **Create a Variation**

  - If the listing is not already in a parentage, you will see **“Create Variations for this Listing.”**

    ![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-WC2TX7U6.png)

    ![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/l6vimage.png)
  - Click this option to begin the process.
4. **Select a Variation Theme**

  - A pop-up will appear prompting you to choose a **Variation Theme** (e.g., Size, Color, etc.).
  - Select the most appropriate theme(s) and click **Continue**.

    ![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-R0GKUVWJ.png)

    ![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/UZrimage.png)
5. **Set Up the Parent Listing**

  - You will be redirected to a form similar to creating a new listing. This will be your **Parent Listing**.
  - Many fields will auto-populate from ASIN1.
  - Fill in all required fields marked in red.
6. **Add Second Child ASIN**

  - After the parent listing is filled, go to the **Variations tab** and manually add the second child listing (**ASIN2**) using the same variation theme.
7. **Finalize**

  - Ensure no red validation boxes remain.
  - Click **Save and Finish**.
8. **Post-Creation Tasks**

  - Wait around **15 minutes** for the parentage to reflect.
  - Then, go back and **restore/edit the child ASIN titles** to their original version if needed (manually or via feed file).

**Best Practices:**

1. Do not reparent if a variation theme is deprecated, until we need to make a change to the variations - taking one away or adding one.

FAQs

**1. What is the difference between "Update" and "PartialUpdate" in the Update_Delete column?**

- Update: Use when creating a new parent listing that doesn’t exist in the account yet.
- PartialUpdate: Use when modifying or adding to an existing parent listing. This ensures only the provided data is updated, leaving the rest untouched.

**2. How should I format the Parent SKU?**

- Create a unique Parent SKU by identifying common elements of the child ASINs and adding "PARENT" to the name. Confirm naming conventions with the client if specific preferences exist.

**3. Why does the Parent SKU title need to be generic?**

- The Parent SKU title may be visible on the product detail page. It should describe the product broadly without referencing specific variations like size, color, or style.

**4. What should I do if the upload status shows "Complete Drafts"?**

- Download the Processing Report to identify and resolve errors. Correct any issues in the flat file and re-upload.

**5. How do I find the correct variation theme?**

- The variation theme depends on the product category. Examples include "Size," "Color," or "Style." Use CTRL + F in the template to locate the corresponding attribute column and enter the variation details for each child SKU.

**6. What should I enter in the Fulfillment Center ID column?**

- FBA (Fulfilled by Amazon): Use AMAZON_NA, AMAZON_EU, etc., based on the marketplace.
- FBM (Fulfilled by Merchant): Use DEFAULT.

**7. How can I verify that the parentage was created successfully?**

- You can confirm parentage through:

  - Manage All Inventory: Search for the Parent SKU and check for attached child SKUs.
  - Variation Wizard: Use the "Add to or update an existing variation family" option to review the parent-child relationship.
  - Amazon Detail Page: Click on the Parent SKU title to ensure all variations appear together.

**8. What should I do if the parent and child ASINs have different brand names?**

- Ensure the brand name matches across all listings. Amazon requires consistency between the parent and child ASINs.

**9. How do I handle compliance details for FBA listings?**

- For FBA products, complete the following fields in the template:

  - Batteries Required
  - Supplier Declared DG HZ Regulation1
  - Pesticide Marking

**10. Can I include both FBA and FBM SKUs in the same parentage?**

- Yes, you can. Ensure both FBA and FBM SKUs are listed in the flat file and correctly associated with the parent SKU.

**11. What are the steps if my flat file upload is not processed correctly?**

- Download the Processing Report to identify errors.
- Correct the data in the flat file (e.g., missing attributes, incorrect variation themes).
- Re-upload the file and monitor the status.

**12. Why do I need to refresh Seller Central before uploading the file?**

- Refreshing ensures you’re on the most up-to-date version of the upload page and prevents submission errors caused by outdated sessions.

**13. How long does Amazon take to process the flat file?**

- Amazon typically processes flat file uploads within 15 minutes to an hour. Monitor the upload status for confirmation.

**14. Can I modify an existing parentage?**

- Yes, use "PartialUpdate" for modifications to prevent overwriting other attributes. Ensure all updates align with the existing parentage setup.

**15. How should I populate the Item Type Keyword?**

- The Item Type Keyword determines the browse classification. Refer to the category-specific template for valid values.
