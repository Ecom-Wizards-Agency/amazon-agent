# Catalog SOP: File Uploads - Delete-Relist

Source: https://sop.myamazonguy.com/books/catalog/page/catalog-sop-file-uploads-delete-relist

## File Uploads - Delete-Relist

**Who is this for:**This is intended for Amazon Sellers and other individuals who are responsible for managing and updating product listings on the Amazon platform.

**Objective:** To provide clear guidance on how to delete and relist a product on Amazon in order to fix any existing listing errors or refresh the listing.

**Overview:** Delete-relist is usually done when we are trying to push for attributes that Amazon somehow is not picking up from our initial upload. This is also used to fix any existing listing error and refresh the listing itself. Below are the most common issues it can fix.

- Category update
- Copy/Search terms update
- Parentage update
- Price Per Unit (PPU) update
- Merging reviews
- Brand Name change

**Overall Process:**

1. Back up data.
2. Prepare the flat file
3. Delete Listing
4. Wait for 24 hours
5. Upload the flat file

**Conditions:**

- Listing has no issues = Do not delete-relist
- Listing has updating/attribute issues > Partial Update did not work = Do not delete-relist. Instead, conduct a full update flat file first. SOP: Full Update Flat File
- Listing has updating/attribute issues > Partial Update did not work > Full Update did not work = Proceed in deleting the listing for 24 hours.

**Next Steps:**

1. Delete-relist can strand an active listing. This means the BM needs to decide if to skip the step because it is too risky or if we can proceed.
2. Comment the reason for the delete-relist on Asana and reassign it back to the Brand Manager.
3. DM the Brand Manager and his/her pod on Slack to notify them about the task that you just reassigned to them. Let them know that you need their confirmation and the client's approval to proceed.
4. The BM needs to reach out to the client and inform the client why we need to do this step and ease their minds on any concerns they may have. (i.e. reviews lost, inventory issues, etc)
5. Ask the BM to reassign the task back to you once they hear back and received a confirmation to proceed from the client.

After a few minutes, head to Inventory and search all SKUs found in the "delete file." If all SKUs have been successfully deleted from the inventory, wait for at least 24 hours before uploading the relist file.

**Upload your Relist File**

19. After at least 24 hours of successfully deleting the listing, ensure that your relist flat file has been QAed and your team leader confirms that you may upload it. Otherwise, keep editing as instructed by your team leader until he/she says it is good to upload.

20. Go to Catalog> Add Products via Upload > Upload your Spreadsheet. Refresh the page first before attaching the file and make sure you’re on the right Amazon account. Then click Upload file.

In the event that you encounter an error during relisting. Create a ticket and request a Catalog Refresh.

You must include the ASINs you wish to update, the issue or attributes you wish to modify, the Batch ID number of the delete template, and the Batch ID number of the relist template.

Note: Deleting and relisting the product with the exact SKU will not affect the existing FBA/FBM inventory (if any), ASIN sales history, or reviews.

FAQS

1. What is a Delete-Relist process?

The Delete-Relist process is used to resolve listing issues that can’t be fixed by partial or full updates. It involves deleting the problematic listing and then uploading a fresh version of it after 24 hours. This process is commonly used to address issues such as category updates, price per unit (PPU) adjustments, brand name changes, or merging reviews.

2. When should I use the Delete-Relist process?

Use the Delete-Relist process when:

- Partial Update and Full Update have not resolved the issues.
- The listing has issues like incorrect category, price, parentage, or brand name that are not being reflected after updates.
- You're troubleshooting complex listing errors that cannot be fixed through simpler methods.

3. When should I not use the Delete-Relist process?

If the listing has no issues or if it can be fixed through Partial Update or Full Update, do not proceed with Delete-Relist.

4. What should I do before deleting the listing?

- Get Brand Manager (BM) approval: Notify the BM about the need to delete and relist and get confirmation from the client.
- Backup data: Download the Category Listings Report (CLR) for backup, which will help preserve all listing data and ensure it's correctly populated during the relist.
- Prepare the relist file: Once you have the CLR, update the relist file with the necessary attributes.

5. How do I prepare the relist file?

1. Download and rename the CLR for the relist process.
2. Ensure the required attributes are filled, especially in the seller contribution box.
3. Check for duplicates in bullet points (bullet 6–10) and remove them.
4. Add images: If applicable, add image URLs manually by copying them from the backend.
5. Change the "Update Delete" column to Update before saving the file.

6. How do I delete the listing?

1. Create a new file: Using a copy of the CLR, delete all unnecessary SKUs and only keep the ones that need to be removed.
2. Set the "Update Delete" column to “Delete” for the SKUs to be deleted.
3. Upload the file: Go to Catalog > Add Products via Upload, and upload the delete file.
4. Check the processing report: Ensure that the delete action was successful. Once confirmed, wait at least 24 hours before proceeding with the relist file.

7. How do I upload the relist file?

After waiting 24 hours:

1. Confirm the file has been QAed by your team leader.
2. Go to Catalog > Add Products via Upload and upload the relist file.
3. Check the processing report: Make sure the file was uploaded successfully.
4. Verify the listing: After the update, check the values on the backend using Ctrl + F (Windows) or Command + F (Mac).

8. Will the Delete-Relist process affect inventory or reviews?

No, deleting and relisting a product with the same SKU will not affect existing FBA/FBM inventory, sales history, or reviews. However, ensure that you follow the correct process to avoid any potential issues.

9. What should I do if I encounter an error during the relisting process?

If you encounter an error after relisting, create a ticket with Amazon Seller Support, providing the ASINs, the issue you're facing, the Batch ID of the delete template, and the Batch ID of the relist template. Request a Catalog Refresh to resolve the issue.

10. Can I delete and relist multiple ASINs at once?

Yes, you can delete and relist multiple ASINs, but make sure that all necessary attributes are updated properly. For larger batches, it’s advisable to check with your team leader before proceeding.
