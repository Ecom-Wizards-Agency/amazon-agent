# Catalog SOP: Full Update Flat File

Source: https://sop.myamazonguy.com/books/catalog/page/catalog-sop-full-update-flat-file

## Full Update Flat File

Who is this for: This is intended for Amazon Sellers and employees who are responsible for updating listings on Amazon.

**Objective:**

The objective of this SOP is to provide clear guidance on the process of conducting a Full Update for listings on Amazon. This includes identifying when a Full Update is necessary, and the specific instructions for conducting a Full Update. The ultimate goal is to ensure accurate and up-to-date product listings for customers and to maintain the competitiveness of the brand on Amazon.

**Overview:** Full update is usually done when a listing has issue/s which include, but are not limited to the following:

1. Category update
2. Copy/Search terms update
3. Parentage update
4. Price Per Unit (PPU) update
5. Merging reviews
6. Brand Name change

**Conditions:**

1. Does the listing have no issues > Full Update? = No
2. Listing needs troubleshooting > Partial Update> Issues weren't solved> Full update? = Yes

**NOTES:**

If a full update flat file has been submitted but the changes are not reflected, proceed with troubleshooting the listing.

- You can submit a ticket to Amazon Seller Support to solve the issue. You can view this [SOP](https://sop.myamazonguy.com/books/catalog/page/catalog-sop-how-to-file-a-ticket-in-seller-central).
- If the issue persists after several attempts, you may proceed with a delete-relist.

FAQs

#### 1. What is a Full Update Flat File?

A Full Update Flat File is used to update a listing’s critical attributes, including category, price, parent-child relationships, brand name, and other product information. It’s typically used when a listing requires major changes or troubleshooting that couldn’t be resolved by a Partial Update.

#### 2. When should I perform a Full Update on a listing?

You should perform a Full Update when:

- The listing needs a category update.
- There are search term or copy updates.
- The parentage (relationship between parent and child ASINs) needs modification.
- There is a Price Per Unit (PPU) update.
- Merging reviews is necessary.
- A brand name change is required.

If no issues exist with the listing, a Full Update is not necessary. If minor troubleshooting is required, a Partial Update is more appropriate.

#### 3. What are the first steps before conducting a Full Update?

- Notify the Brand Manager (BM) via Slack DM, explaining why the full update is necessary.
- The BM’s approval is required before proceeding. If the BM approves, proceed with the update; if they deny, note the reason and reassign the task back to the BM.

#### 4. How do I prepare the Full Update flat file?

1. Download the Category Listings Report (CLR) from Seller Central:

  - Go to Inventory > Inventory Reports.
  - Select Category Listings Report and click Request Report.
2. Once the report is generated, download it and create a copy.
3. Open the report and enable editing.
4. Remove unnecessary SKUs: Keep only the SKUs you need to update.
5. Ensure the correct fields are filled:

  - Seller contribution box: Copy the information from the top of the box, not the empty data inside it.
  - Sales price: If missing, manually add this data to avoid overriding existing pricing.
6. Remove duplicate bullet points: Check bullets 6–10 for any duplicates and remove them.
7. Ensure correct unit count: This will ensure the PPU is displayed correctly on the product page.
8. Images:

  - If working with EU listings, be extra cautious as image URLs may disappear during Full Update. If necessary, add URLs for the main image and other images.
9. Change the “Update Delete” column to Update and save the file.

#### 5. How do I upload the Full Update file to Seller Central?

- Go to Catalog > Add Products via Upload in Seller Central.
- Refresh the page and ensure you are on the correct Amazon store.
- Attach and upload the updated file.

#### 6. How long does it take for the Full Update to reflect on the listing?

It may take some time (up to 24 hours) for changes to reflect on the product detail page (PDP) or in the backend. To verify if the update is successful:

- Use Ctrl + F (Windows) or Command + F (Mac) to search for the updated values in the seller contribution box.

#### 7. What if the Full Update does not work?

If the changes do not reflect, troubleshoot the listing and ensure that the correct file was uploaded. If the issue persists:

- Submit a ticket to Amazon Seller Support for assistance.
- If repeated attempts fail, consider doing a delete-relist to correct the listing.

#### 8. Can I update multiple listings at once using the Full Update flat file?

Yes, you can update multiple listings in one file, but ensure that each SKU is updated accurately and that no unnecessary SKUs are included.

#### 9. What are the risks of using a Full Update flat file?

The main risk of using a Full Update is that changes may not reflect properly, especially in the case of product images, and the update could overwrite certain values if not handled carefully.
