# Catalog SOP: A/B Test (Bullet Points)

Source: https://sop.myamazonguy.com/books/catalog/page/catalog-sop-ab-test-bullet-points

## A/B Test (Bullet Points)

Who is this for? Individuals who wished to conduct A/B split testing within Seller Central

Objective: The purpose of this document is to provide a step-by-step guide on how to create A/B split testing to submit alternate content for your products and collect insights such as traffic and conversion.

Split testing or AB testing on Amazon, is an excellent way of optimizing your listings through experiments with product content, images, and pricing. Four crucial elements of the listing affect conversion rate: Image, Title, Product Features, Description, and Price.

**Seller Central A/B Testing**

Access the Amazon Seller Central page by clicking the URL: [https://sellercentral.amazon.com/](https://sellercentral.amazon.com/)

- Go to **Brands**, then select **Manage Experiments**.

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-JQDE43U1.png)

[![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/5bcimage.png)](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/5bcimage.png)

- On the experiments dashboard, click Create a New Experiment.

  - You can run an experiment for **A+ Content, Bullet Points, Main Image, Product Description, and Product Title.**

**Bullet Points A/B Testing**

- If the ASIN is part of a variation, you will need to select first the whole parentage as the reference ASIN.
- Enter a descriptive test name, duration, and desired start date. You may also enter an optional hypothesis around what you wish to learn from the test.

  - Experiment Name
  - Hypothesis - Brief description of the experiment and the desired outcome.
  - Duration - Minimum of 4 weeks to a maximum of 10 weeks.

    - Start Date will be automatically set 1 week after you created the experiment, but you can still set it at a later date.
    - End Date will be automatically set depending on the duration of the experiment.
  - You can tick To significance which automatically ends the experiment when there is enough time data to declare winner.
  - You can also set it to Automatically publish once the experiment is done.

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/image-DUWYMZK8.png)

[![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/iS8image.png)](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/iS8image.png)

- Uncheck all other child ASINs that are not part of the experiment.

  - When you run an experiment on a variation family, you can submit an experiment for one or more of the child ASINs, but you always have to submit for the parent ASIN.
- Type in the new bullet points in the Version B section of the ASIN involved.

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/catalog-sop-ab-test-bullet-points-image-3.jpg)

- Once all necessary fields are filled up, click Schedule Experiment.
- Click View Details to check the experiment's current status.

FAQs

**1. Who is eligible to run A/B tests in Seller Central?**

- A/B testing is available only to brand-registered sellers who own the ASINs they are testing. If you don’t see the option to run an experiment, check if your ASIN is brand-registered and if it qualifies for experiments.

**2. How do I interpret the results of my A/B test?**

- Amazon provides data on conversion rate, sales, and traffic for each version. If one version shows a statistically significant improvement, it's the better option. If results are inconclusive, the test may need more time or a more substantial change.

**3. What if my listing doesn’t get enough traffic for a conclusive result?**

- Low-traffic ASINs may struggle to reach statistical significance. To improve results:

  - Extend the test duration to collect more data.
  - Prioritize testing high-traffic ASINs first.
  - Consider making more noticeable changes in bullet points.

**4. What are best practices for testing bullet points?**

- Test one element at a time (e.g., benefit-driven vs. feature-driven text).
- Make meaningful changes—small tweaks may not generate significant results.
- Keep structure consistent (same number of bullet points in both versions).
- Use data-driven insights (e.g., analyze competitor listings for trends).

**5. Can I edit or cancel an experiment after launching it?**

- No, once an experiment is scheduled, it cannot be edited. You’ll need to stop the current experiment and create a new one if changes are needed.

**6. What happens if both versions perform similarly with no clear winner?**

- If there’s no significant difference, consider testing a larger change (e.g., completely restructuring bullet points or focusing on stronger benefit-driven language). If neither version outperforms the other, stick with the version that aligns better with your brand messaging.

**7. How long does it take for changes to reflect after an experiment is published?**

- Once an experiment is completed and a version is selected for publication, changes can take 24–48 hours to reflect on the live product detail page.

**8. What should I do if I encounter issues setting up an A/B test?**

- Ensure your ASIN is brand-registered and eligible for experiments.
- Check if the ASIN is already part of an ongoing A/B test.
- Refresh the page, clear cache, or try a different browser.
- If issues persist, contact Amazon Seller Support for troubleshooting.
