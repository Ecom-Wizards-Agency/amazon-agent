# Catalog SOP: Parentage Check

Source: https://sop.myamazonguy.com/books/catalog/page/catalog-sop-parentage-check

## **Parentage Check**

**Who is this for :** Individuals desiring to perform a parentage check within their selling account.

**Objective:** To facilitate individuals in acquiring the necessary knowledge to conduct a Parentage Check, gain an understanding of the parentage model, assign orphaned duplicate SKUs to their corresponding parent ASIN, and optimize the title, bullets, product descriptions, and search terms of the parent ASIN, thereby streamlining the catalog and improving SEO prospects.

**Orphaned SKU (Stock Keeping Unit) :**

- A SKU (Stock Keeping Unit) that is no longer associated with an active or valid product listing in the Amazon catalog.
- A SKU that exists in your inventory or system but is disconnected from its corresponding product listing.

**Causes of Orphaned SKUs:**

1. Listing Deletion:

- The product listing associated with the SKU has been deleted or deactivated.

1. Listing Suppression:

- The listing is suppressed by Amazon due to policy violations, missing information, or quality issues (e.g., missing images, titles, or descriptions).

1. SKU Migration Errors:

- During catalog merges or updates, the SKU is detached from its original product listing.

1. Discontinued Products:

- Products that are no longer sold but still have SKUs in your inventory.

1. Errors in Inventory Upload:

- Incomplete or incorrect bulk inventory uploads can leave SKUs unattached.

1. Fulfillment Center Issues:

- If inventory is moved or mishandled, the SKU might be left in the system without an associated active listing.

**Goals:** Parentage Check is a proactive task we conduct at least once every 3 months to:

1. To identify if the parentage is a **Display Parent Model** or a **Display Child Model**

  - Display Parent Model (DPM) means that the title displayed will be that of the parent variation, regardless of which variation is chosen
  - Display Child Model (DCM) displays the product title using the text from the child within the variation.
  - Regardless of the parentage model, the parent product title must encompass all the children's listings in a variation.
2. Check for orphaned duplicate SKUs
3. Parent any orphaned duplicate SKU according to its assigned parent ASIN

  - Doing this will clean up the catalog
4. For **Display Parent Model Listings,** under the categories **Clothing, Shoes, Sports, Jewelry and Outdoor**, ensure parent ASINs have basic, informative content that provides an overview of the product line, without the need for extensive optimization :

  - For SEO Opportunities

    - Parent ASIN title must be optimized according to MAG standards
  - Extra steps (not necessary for SEO optimizations) :

    - Parent bullets must be filled up
    - Parent product descriptions and search terms must be filled up

      Other reasons why we need to conduct a parentage check: To find other potential parentage setups

5. Report any other errors you see along the way.

Part 1

##### **PART 1**

**HOW TO IDENTIFY DISPLAY PARENT MODEL VS. DISPLAY CHILD MODEL**

Product titles assigned to each child listing are visible on the product listing. To check and confirm if a variation follows the DCM, follow these steps:

1. Go to the product Amazon detail page by searching [amazon.com](https://www.amazon.com/dp/) in your browser. ![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-1.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/yqiimage.png)

2. Get the parent or child ASIN and search it on the browser by following this format: [https://www.amazon.com/dp/](https://www.amazon.com/dp/)ASIN

3. Click each child variation. If the titles change as you select a new variant, you can confirm that the listing follows the DCM.![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-2.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/xpWimage.png)

A Variation that follows a DPM, on the other hand, will simply display the product title we assign to the Parent ASIN. As you can see in the sample below, the title remains unchanged even if we select the other variants on the parentage.

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/I2Cimage.png)

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-3.png)

**HOW TO IDENTIFY ORPHANED DUPLICATE SKUS**

1. Search the parent ASIN on Seller Central Inventory Page

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-4.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/aC0image.png)

2. Check the number of the child-ASINs.![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-5.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/jGMimage.png)

3. Next, go to the Variation Wizard and pull a Variation Report by following this step: Menu > Catalog > Add a Product > Create Variations Using Variation Wizard

NOTE: Once you click the hyperlink Create Variations Using Variation Wizard, you will be routed to "Variation Wizard."

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/N2Oimage.png)

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-6.png)

4. Click on the first option "Add to or update an existing variation family" then input your parent ASIN

![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-7.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/kFOimage.png)

5. Count the number of child SKUs on the Variation Wizard![image](https://files.document360.io/d9e9b7a2-6758-4c75-97d3-36a6877048cf/Images/Documentation/Catalog-SOP-Parentage-Check-Part-1-8.png)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/Kgfimage.png)

6. If the number of children SKUs is not the same as the ones in the inventory, this means you will need to do a parentage clean-up and parent the duplicate listings to their correct parent ASINs.

**Other things you need to check for:**

- Variation Theme must be present on all SKUs in the VW Report
- All child ASINs appear under one product detail page.
- Variation theme-related attributes are accurate on the product detail page.

FAQs

1. What is an orphaned SKU?

An orphaned SKU is a stock-keeping unit (SKU) that exists in your inventory but is not linked to an active product listing in Amazon’s catalog.

2. What causes SKUs to become orphaned?

- Listing Deletion or Suppression – The listing was removed due to deactivation, policy violations, or missing information.
- SKU Migration Errors – The SKU was detached during catalog updates.
- Discontinued Products – The product is no longer sold, but the SKU remains in inventory.
- Inventory Upload Errors – Incorrect bulk uploads left SKUs unattached.
- Fulfillment Center Issues – Inventory was moved or mishandled, leaving the SKU unlinked.

3. How often should a parentage check be performed?

At least once every three months to maintain catalog accuracy and improve SEO.

4. What are the key objectives of a parentage check?

- Identify whether the listing follows Display Parent Model (DPM) or Display Child Model (DCM).
- Locate and reassign orphaned duplicate SKUs.
- Ensure parent ASINs contain all relevant child listings.
- Optimize parent titles, bullet points, descriptions, and search terms for SEO.
- Verify variation themes and correct errors.

5. How do I check if a listing follows a Display Parent Model (DPM) or Display Child Model (DCM)?

1. Go to the product detail page on Amazon.
2. Search for the parent or child ASIN using: https://www.amazon.com/dp/ASIN.
3. Click on different variations:

  - If the title remains the same, it follows DPM.
  - If the title changes, it follows DCM.

6. How do I identify orphaned duplicate SKUs?

1. Search the parent ASIN in Seller Central > Inventory Page.
2. Count the number of child ASINs.
3. Go to Catalog > Variation Wizard > Add to or update an existing variation family.
4. Enter the parent ASIN and compare the child ASIN count with the Inventory Page.
5. If the numbers differ, reassign orphaned SKUs to their correct parent.

7. How do I reassign orphaned SKUs?

1. Create a parentage flat file following Amazon’s SOP.
2. Ensure the correct Parent SKU and Variation Theme are applied.
3. Submit the file via Inventory Loader or Flat File Upload.

8. How do I check for SEO optimization in parent ASINs?

1. Download the Category Listings Report from Inventory Reports.
2. Filter for the Parent SKU to view associated child SKUs.
3. Review titles, bullet points, descriptions, and search terms for completeness.
4. Update the parent ASIN’s content to ensure proper optimization.
