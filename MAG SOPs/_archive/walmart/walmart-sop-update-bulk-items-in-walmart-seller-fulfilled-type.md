## **Update Bulk Items in Walmart (Seller Fulfilled Type)**

**Who is this for :**For various teams or individuals responsible for updating the content on the Walmart platforms.

**Objective :**Successfully update specific attributes of bulk listings while reducing errors and turnaround time.

**Instructions:**

1. Log in to Walmart seller central account. Go to [https://seller.walmart.com/](https://seller.walmart.com/)

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/Adbimage.png)

2. Navigate to the **Catalog**in the main menu, enter the **SKU**in the search bar, and check the **Category.**

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/ZsSimage.png)

- If the Category is not visible in the line item, click ***Columns*** and tick the box for the Category option and hit **Apply**to add it to the line item.

5. You will be redirected to the **Update All Attributes** page. Choose the correct categories for the items you want to update and click **Next.**

1. Select the product type for the items you want to update. You can choose up to 10 types and hit **Next.**

1. On the **Review Summary and Download** page, double-check that you selected the correct **Category** and **Product Type**. For **Fulfillment**, choose **Seller Fulfilled**, then click **Download**.

1. Once the category maintenance spec sheet is downloaded, open the file and provide the **SKU**, **product type** and **product identifier. You** can remove the rest of the attributes or update them based on your needs.

![image.png](https://sop.myamazonguy.com/uploads/images/gallery/2025-08/scaled-1680-/qeUimage.png)

1. After filling out the template, navigate back to your **Catalog** in the Seller Center. Select **Update Items** and choose **Update with File** from the dropdown menu. A success message will appear displaying the **Feed ID.** Updates may take as little as 15 minutes or up to four hours to appear.

Select the **Update All Attributes** file type, choose your preferred fulfillment type, and upload your file. A success message will appear displaying the **Feed ID.** Updates may take as little as 15 minutes or up to four hours to appear.

FAQs

1. When should I use bulk updates instead of updating items individually?

Use bulk updates when you need to update multiple SKUs for attributes like:

- Pricing, inventory levels, fulfillment type, or category
- Content updates (titles, descriptions, key features, images)
- Product attributes (dimensions, weight, materials, colors)

Use single item updates if you only need to modify a few listings or if Walmart locks certain attributes that require a case submission.

2. What are the most common bulk update errors, and how do I fix them?

If Walmart rejects your bulk upload, check the error report in the Activity Feed. Common errors include:

- Invalid SKU → Ensure the SKU matches exactly as listed in Walmart Seller Center.
- Incorrect Product Category → Confirm that the selected category matches Walmart’s taxonomy.
- Missing Required Fields → Ensure you do not delete mandatory columns from the spec sheet.
- Restricted Updates → Some attributes (e.g., UPC/GTIN) cannot be changed; submit a support case instead.

3. How do I correctly update specific attributes in bulk?

- Titles & Descriptions → Update these fields in the CSV file under Product Name, Short Description, and Key Features.
- Images → Include a direct URL link to the new image in the Main Image URL column.
- Pricing & Inventory → Modify Price, Selling Price, and Available Quantity fields.
- Category Changes → If the product is misclassified, select the correct category from Walmart’s taxonomy and ensure it matches your product type.

4. Can I change any product attribute using bulk updates?

No, certain attributes cannot be changed once a product is created, including:

- GTIN/UPC – Cannot be modified; a new listing is required.
- Brand Name – Locked after the first submission unless Walmart support intervenes.
- Manufacturer Part Number – May be locked for certain categories.

For locked fields, submit a case to Walmart Partner Support instead of a bulk update.

5. What should I do if my product is in the wrong category?

1. Check the current category by downloading an Item Report in Seller Center.
2. Use Walmart’s Category Taxonomy Guide to find the correct category.
3. Update the Category Path using a bulk update template.
4. If Walmart rejects the update, submit a Category Path Update Request via Partner Support.

6. How long does it take for bulk updates to process?

- Basic updates (price, inventory, fulfillment type) → 15 minutes to 2 hours.
- Content updates (titles, descriptions, images) → 2 to 4 hours.
- Category updates → May take up to 24 hours to fully reflect on Walmart.com.

7. How do I verify that my updates were successfully processed?

- Go to Activity Feed > Search by Feed ID to check status.
- If the status is "Processed with Errors", download the error report.
- If the update was successful, check the Item Report or click "View on Walmart.com" to confirm changes.

8. What is a Feed ID, and why is it important?

The Feed ID is a unique identifier for every bulk upload.

- It helps track whether Walmart successfully processed the update.
- If Walmart rejects an update, the Feed ID allows you to pull an error report.
- You can use it to escalate an issue with Walmart support if the update does not reflect.

9. What should I do if my bulk update does not reflect after 24 hours?

1. Check the Activity Feed to confirm if Walmart processed the feed successfully.
2. Download the Item Report to see if the changes are recorded in Walmart’s system.
3. Clear your browser cache and check if the Walmart.com listing has refreshed.
4. If the update is still missing, submit a Partner Support case with the Feed ID for escalation.

10. How many SKUs should I update in one bulk upload?

To prevent file processing errors, follow these batch size guidelines:

- Small updates (price, inventory) → Up to 5,000 SKUs per file.
- Content updates (titles, descriptions, images) → Up to 1,000 SKUs per file.
- Category or attribute updates → Up to 500 SKUs per file to avoid rejection.

For larger updates, break the file into smaller batches to reduce errors.
