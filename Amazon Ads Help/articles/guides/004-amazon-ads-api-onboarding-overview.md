# Amazon Ads API onboarding overview

The Amazon Ads API is used to manage advertising resources for an Amazon advertiser through a REST API. Direct advertisers, partners, and integrators can apply for access.

Approved applicants establish a client application through Login with Amazon (LwA). That client application becomes the credentialed application used for Ads API requests.

## Onboarding flow

| Step | Action | Notes |
| --- | --- | --- |
| 1 | Create a Login with Amazon application | The LwA client application is free to create and does not require approval. |
| 2 | Apply for Amazon Ads API access | Application includes business-use questions plus review of the Amazon Ads API License Agreement and Data Protection Policy. |
| 3 | Assign API access to the LwA application | After approval, enable the LwA client application to make Ads API requests. |

## Important checkpoints

- Application approval can take up to 1 business day.
- Start with the LwA application before applying access to the API.
- If connection issues happen later, use Amazon's technical support page for API support.
